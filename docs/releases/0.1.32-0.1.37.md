# Release Notes: 0.1.32 - 0.1.37

## Highlights

This phase expanded the configuration surface, removed remaining hardcoded comment/about UI assumptions, and tightened release safety around starter/package coordination.

## Major Themes

### About page runtime hardening

- made about runtime validation more config-driven
- localized about content by locale instead of assuming one shared object
- removed remaining hardcoded labels from about UI paths

### Pagination and archive configuration

- componentized config-driven pagination styles
- fixed random pagination variant parsing issues

### Workflow and safety improvements

- enforced maintainer-only starter sync boundaries
- clarified check scripts and workspace safety guards
- migrated RSS endpoint handling to TypeScript
- documented npm release gates and starter sync manifest rules

### Comments and configuration surface

- made comments configurable instead of hardcoded
- completed Giscus mapping and i18n cleanup work
- exposed full Giscus attributes through theme config so users could fully control embedded discussion behavior

## Upgrade Impact

- users gained a materially better configuration surface, especially for comments and localized about content
- maintainers gained stronger safety rails around workspace links, release gates, and starter synchronization
