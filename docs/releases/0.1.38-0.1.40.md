# Release Notes: 0.1.38 - 0.1.40

## Highlights

This phase closed release workflow gaps, hardened comments and metrics behavior, and introduced the current document metadata/agent workflow system.

## Major Themes

### Comment and runtime correctness

- validated Giscus number mapping more defensively
- localized remaining hardcoded blog-post system labels
- hardened metrics counting and comment integration behavior

### Release and starter synchronization

- made starter sync single-source from `main`
- sanitized push/sync flows
- closed dependency restoration and starter drift gaps in the release workflow

### Documentation and agent workflow system

- unified agent entrypoints and release guidance
- replaced the fake frontmatter parser with `gray-matter`
- formalized the document metadata contract and adapter graph

## Upgrade Impact

- theme consumers received correctness improvements in comments and metrics by `0.1.40`
- maintainers received a more reliable release/starter sync process and a formalized documentation workflow system
