# Release Notes: 0.1.25 - 0.1.31

## Highlights

This phase focused on packaging quality, starter identity cleanup after the org rename, richer demo content, and more advanced blog-list pagination behavior.

## Major Themes

### Packaging and tooling reliability

- added a unified npm publish command
- introduced Husky and lint-based engineering gates
- fixed published package contents to include scaffold files and tarball CLI safety checks

### Repository and starter identity cleanup

- migrated remaining repo/template references from VoidTemple to Anglefeint
- removed outdated example comments and stale org references

### Starter/demo content expansion

- added four themed posts across all locales to better demonstrate the route-based visual system
- refined `hello-world` tone and aligned sample content with the project direction

### Blog list experience

- introduced windowed cyber pagination with per-item style variation
- continued tightening pagination behavior and surface polish

### About page polish

- switched about sidebar hover accenting away from duplicated homepage green
- enriched AI modal status copy and runtime presentation

## Upgrade Impact

- no hard migration required for normal users
- package consumers gained a more complete scaffold and more representative starter/demo content
- maintainers gained a more usable publish and lint workflow during this phase
