# Release Notes: 0.1.16 - 0.1.24

## Highlights

This phase established the Astro 6 baseline, hardened scaffolding and locale/config contracts, and then concentrated on blog-post performance and Red Queen playback stability.

## Major Themes

### Astro 6 and platform alignment

- aligned Node and Astro requirements for the Astro 6 beta track
- migrated schema imports to `astro/zod`
- updated i18n/default-override wiring and documentation

### Scaffold and configuration hardening

- strengthened `new-page` and `new-post` generation with tests and checks
- introduced config-driven locales, doctor flow improvements, and Red Queen effect toggles
- documented adapter sync and release workflow contracts more explicitly

### Post-detail performance and UX

- decomposed AI/About styles into layered partials
- moved critical detail-page effects toward composited animation paths and better asset caching
- stabilized LCP by making hero rendering static-first before deferred preparation
- reworked Red Queen playback timing, preload, and autoplay gating to reduce duplicate fetches and heavy startup cost

### Operational additions

- added font asset checks into the validation pipeline
- synchronized Red Queen lifecycle documentation with actual runtime behavior

## Upgrade Impact

- no known destructive migration for normal theme consumers
- users on older Astro stacks needed to match the documented Astro 6 beta baseline during this phase
- post-detail behavior became more stable without removing the visual identity of the theme
