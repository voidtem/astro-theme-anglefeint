# Release Notes: 0.2.5

## Highlights

This release finishes the terminology cleanup around the single-source locale registry. The starter/runtime adapter surface now reads like the current model instead of carrying older naming conventions forward.

## Major Themes

### Locale naming cleanup

- renamed locale exports from `SUPPORTED_LOCALES` to `ENABLED_LOCALES`
- renamed locale label exports from `LOCALE_LABELS` to `ENABLED_LOCALE_LABELS`
- updated all starter-side routes, RSS generation, SEO alternate-link generation, and starter templates to the new naming

### Site adapter cleanup

- removed `SITE_HERO_BY_LOCALE` from the starter/package adapter surface
- standardized on `getSiteHero(locale)` as the site-hero selector
- renamed the About toggle adapter constant from `ENABLE_ABOUT_PAGE` to `ABOUT_PAGE_ENABLED`

### Tooling and fixture alignment

- removed the stale `meta.dir` residue from the i18n integration fixture
- updated adapter contract checks and scaffold checks to assert against the new selector/export names
- aligned package-internal defaults and starter adapter templates so installed-package users and starter users see the same naming model

## Migration Notes

- if your project imports starter/runtime adapter symbols directly, update:
  - `SUPPORTED_LOCALES` -> `ENABLED_LOCALES`
  - `LOCALE_LABELS` -> `ENABLED_LOCALE_LABELS`
  - `SITE_HERO_BY_LOCALE[locale]` -> `getSiteHero(locale)`
  - `THEME.ENABLE_ABOUT_PAGE` -> `THEME.ABOUT_PAGE_ENABLED`
- no content migration is required
- starter sync is required for this release because starter-distributed adapter files and templates changed

## Starter Sync

- starter sync is required for this release
