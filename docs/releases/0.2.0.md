# Release Notes: 0.2.0

## Highlights

This release establishes the new baseline architecture for Anglefeint: a single-source i18n configuration model, locale-aware scaffolding, configurable default-locale routing, and a lightweight browser smoke-test layer.

## Major Themes

### Single-source i18n architecture

- moved starter locale ownership into `src/site.config.ts -> i18n`
- replaced fixed-locale assumptions with structured locale metadata, fallback chains, localized UI overrides, per-locale hero copy, and per-locale About config
- unified route generation, header language options, and SEO alternate metadata around the same normalized locale registry

### Routing and SEO correctness

- completed `i18n.routing.defaultLocalePrefix` support for both `'never'` and `'always'`
- aligned homepage redirects, canonical URLs, hreflang generation, Open Graph locale metadata, and sitemap filtering with the selected default-locale routing mode

### Scaffolding and starter workflow

- updated `new-post` to resolve locales from the configured starter registry instead of a fixed package locale list
- added support for region-code locales and safer fallback template generation for unknown locales
- kept `new-page` and adapter sync aligned with the new starter/runtime contract

### Validation and smoke testing

- added a minimal Playwright smoke-test layer with `npm run e2e:install` and `npm run e2e`
- covered homepage default-locale behavior, head alternate metadata, and language switcher behavior in a real browser path

## Migration Notes

- this release is intended to serve as the new baseline package/starter contract
- starter consumers should use the new `src/site.config.ts -> i18n` structure rather than the older split locale fields

## Starter Sync

- starter sync is required for this release because the package/runtime contract and starter-side config surface changed together

## Validation Summary

- `npm run check`
- `npm run e2e`
